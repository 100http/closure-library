FastDataNode.fromJs(object, dataName, opt_parent)code »

Creates a new FastDataNode structure initialized from object. This will return an instance of the most suitable sub-class of FastDataNode.

You should not modify object after creating a fast data node from it or assume that changing object changes the data node. Doing so results in undefined behaviour.

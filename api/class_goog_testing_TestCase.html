class TestCase

A class representing a JsUnit test case. A TestCase is made up of a number of test functions which can be run. Individual test cases can override the following functions to set up their test environment: - runTests - completely override the test's runner - setUpPage - called before any of the test functions are run - tearDownPage - called after all tests are finished - setUp - called before each of the test functions - tearDown - called after each of the test functions - shouldRunTests - called before a test run, all tests are skipped if it returns false. Can be used to disable tests on browsers where they aren't expected to pass.

Use #autoDiscoverLifecycle and #autoDiscoverTests

runTests()code »

Executes each of the tests, yielding asynchronously if execution time exceeds #maxRunTime. There is no guarantee that the test case has finished execution once this method has returned. To be notified when the test case has finished execution, use #setCompletedCallback or #runTestsReturningPromise.

Overridable by the individual test case. This allows test cases to defer when the test is actually started. If overridden, finalize must be called by the test to indicate it has finished.

runTestsReturningPromise()code »

Executes each of the tests, returning a promise that resolves with the test results once they are done running.

15/**
16 * @fileoverview Browser history stack management class.
17 *
18 * The goog.History object allows a page to create history state without leaving
19 * the current document. This allows users to, for example, hit the browser's
20 * back button without leaving the current page.
21 *
22 * The history object can be instantiated in one of two modes. In user visible
23 * mode, the current history state is shown in the browser address bar as a
24 * document location fragment (the portion of the URL after the '#'). These
25 * addresses can be bookmarked, copied and pasted into another browser, and
26 * modified directly by the user like any other URL.
27 *
28 * If the history object is created in invisible mode, the user can still
29 * affect the state using the browser forward and back buttons, but the current
30 * state is not displayed in the browser address bar. These states are not
31 * bookmarkable or editable.
32 *
33 * It is possible to use both types of history object on the same page, but not
34 * currently recommended due to browser deficiencies.
35 *
36 * Tested to work in:
37 * <ul>
38 * <li>Firefox 1.0-4.0
39 * <li>Internet Explorer 5.5-9.0
40 * <li>Opera 9+
41 * <li>Safari 4+
42 * </ul>
43 *
44 * @author brenneman@google.com (Shawn Brenneman)
45 * @see ../demos/history1.html
46 * @see ../demos/history2.html
47 */
48
49/* Some browser specific implementation notes:
50 *
51 * Firefox (through version 2.0.0.1):
52 *
53 * Ideally, navigating inside the hidden iframe could be done using
54 * about:blank#state instead of a real page on the server. Setting the hash on
55 * about:blank creates history entries, but the hash is not recorded and is lost
56 * when the user hits the back button. This is true in Opera as well. A blank
57 * HTML page must be provided for invisible states to be recorded in the iframe
58 * hash.
59 *
60 * After leaving the page with the History object and returning to it (by
61 * hitting the back button from another site), the last state of the iframe is
62 * overwritten. The most recent state is saved in a hidden input field so the
63 * previous state can be restored.
64 *
65 * Firefox does not store the previous value of dynamically generated input
66 * elements. To save the state, the hidden element must be in the HTML document,
67 * either in the original source or added with document.write. If a reference
68 * to the input element is not provided as a constructor argument, then the
69 * history object creates one using document.write, in which case the history
70 * object must be created from a script in the body element of the page.
71 *
72 * Manually editing the address field to a different hash link prevents further
73 * updates to the address bar. The page continues to work as normal, but the
74 * address shown will be incorrect until the page is reloaded.
75 *
76 * NOTE(user): It should be noted that Firefox will URL encode any non-regular
77 * ascii character, along with |space|, ", <, and >, when added to the fragment.
78 * If you expect these characters in your tokens you should consider that
79 * setToken('<b>') would result in the history fragment "%3Cb%3E", and
80 * "esp&eacute;re" would show "esp%E8re". (IE allows unicode characters in the
81 * fragment)
82 *
83 * TODO(user): Should we encapsulate this escaping into the API for visible
84 * history and encode all characters that aren't supported by Firefox? It also
85 * needs to be optional so apps can elect to handle the escaping themselves.
86 *
87 *
88 * Internet Explorer (through version 7.0):
89 *
90 * IE does not modify the history stack when the document fragment is changed.
91 * We create history entries instead by using document.open and document.write
92 * into a hidden iframe.
93 *
94 * IE destroys the history stack when navigating from /foo.html#someFragment to
95 * /foo.html. The workaround is to always append the # to the URL. This is
96 * somewhat unfortunate when loading the page without any # specified, because
97 * a second "click" sound will play on load as the fragment is automatically
98 * appended. If the hash is always present, this can be avoided.
99 *
100 * Manually editing the hash in the address bar in IE6 and then hitting the back
101 * button can replace the page with a blank page. This is a Bad User Experience,
102 * but probably not preventable.
103 *
104 * IE also has a bug when the page is loaded via a server redirect, setting
105 * a new hash value on the window location will force a page reload. This will
106 * happen the first time setToken is called with a new token. The only known
107 * workaround is to force a client reload early, for example by setting
108 * window.location.hash = window.location.hash, which will otherwise be a no-op.
109 *
110 * Internet Explorer 8.0, Webkit 532.1 and Gecko 1.9.2:
111 *
112 * IE8 has introduced the support to the HTML5 onhashchange event, which means
113 * we don't have to do any polling to detect fragment changes. Chrome and
114 * Firefox have added it on their newer builds, wekbit 532.1 and gecko 1.9.2.
115 * http://www.w3.org/TR/html5/history.html
116 * NOTE(goto): it is important to note that the document needs to have the
117 * <!DOCTYPE html> tag to enable the IE8 HTML5 mode. If the tag is not present,
118 * IE8 will enter IE7 compatibility mode (which can also be enabled manually).
119 *
120 * Opera (through version 9.02):
121 *
122 * Navigating through pages at a rate faster than some threshhold causes Opera
123 * to cancel all outstanding timeouts and intervals, including the location
124 * polling loop. Since this condition cannot be detected, common input events
125 * are captured to cause the loop to restart.
126 *
127 * location.replace is adding a history entry inside setHash_, despite
128 * documentation that suggests it should not.
129 *
130 *
131 * Safari (through version 2.0.4):
132 *
133 * After hitting the back button, the location.hash property is no longer
134 * readable from JavaScript. This is fixed in later WebKit builds, but not in
135 * currently shipping Safari. For now, the only recourse is to disable history
136 * states in Safari. Pages are still navigable via the History object, but the
137 * back button cannot restore previous states.
138 *
139 * Safari sets history states on navigation to a hashlink, but doesn't allow
140 * polling of the hash, so following actual anchor links in the page will create
141 * useless history entries. Using location.replace does not seem to prevent
142 * this. Not a terribly good user experience, but fixed in later Webkits.
143 *
144 *
145 * WebKit (nightly version 420+):
146 *
147 * This almost works. Returning to a page with an invisible history object does
148 * not restore the old state, however, and there is no pageshow event that fires
149 * in this browser. Holding off on finding a solution for now.
150 *
151 *
152 * HTML5 capable browsers (Firefox 4, Chrome, Safari 5)
153 *
154 * No known issues. The goog.history.Html5History class provides a simpler
155 * implementation more suitable for recent browsers. These implementations
156 * should be merged so the history class automatically invokes the correct
157 * implementation.
158 */

188/**
189 * A history management object. Can be instantiated in user-visible mode (uses
190 * the address fragment to manage state) or in hidden mode. This object should
191 * be created from a script in the document body before the document has
192 * finished loading.
193 *
194 * To store the hidden states in browsers other than IE, a hidden iframe is
195 * used. It must point to a valid html page on the same domain (which can and
196 * probably should be blank.)
197 *
198 * Sample instantiation and usage:
199 *
200 * <pre>
201 * // Instantiate history to use the address bar for state.
202 * var h = new goog.History();
203 * goog.events.listen(h, goog.history.EventType.NAVIGATE, navCallback);
204 * h.setEnabled(true);
205 *
206 * // Any changes to the location hash will call the following function.
207 * function navCallback(e) {
208 * alert('Navigated to state "' + e.token + '"');
209 * }
210 *
211 * // The history token can also be set from code directly.
212 * h.setToken('foo');
213 * </pre>
214 *
215 * @param {boolean=} opt_invisible True to use hidden history states instead of
216 * the user-visible location hash.
217 * @param {!goog.html.TrustedResourceUrl|string=} opt_blankPageUrl A URL to a
218 * blank page on the same server. Required if opt_invisible is true. If
219 * possible pass a TrustedResourceUrl; string is supported for
220 * backwards-compatibility only and uses goog.html.legacyconversions.
221 * This URL is also used as the src for the iframe used to track history
222 * state in IE (if not specified the iframe is not given a src attribute).
223 * Access is Denied error may occur in IE7 if the window's URL's scheme
224 * is https, and this URL is not specified.
225 * @param {HTMLInputElement=} opt_input The hidden input element to be used to
226 * store the history token. If not provided, a hidden input element will
227 * be created using document.write.
228 * @param {HTMLIFrameElement=} opt_iframe The hidden iframe that will be used by
229 * IE for pushing history state changes, or by all browsers if opt_invisible
230 * is true. If not provided, a hidden iframe element will be created using
231 * document.write.
232 * @constructor
233 * @extends {goog.events.EventTarget}
234 */
235goog.History = function(opt_invisible, opt_blankPageUrl, opt_input,
236 opt_iframe) {
237 goog.events.EventTarget.call(this);
238
239 if (opt_invisible && !opt_blankPageUrl) {
240 throw Error('Can\'t use invisible history without providing a blank page.');
241 }
242
243 var input;
244 if (opt_input) {
245 input = opt_input;
246 } else {
247 var inputId = 'history_state' + goog.History.historyCount_;
248 var inputHtml = goog.html.SafeHtml.create('input',
249 {type: goog.dom.InputType.TEXT, name: inputId, id: inputId,
250 style: goog.string.Const.from('display:none')});
251 goog.dom.safe.documentWrite(document, inputHtml);
252 input = goog.dom.getElement(inputId);
253 }
254
255 /**
256 * An input element that stores the current iframe state. Used to restore
257 * the state when returning to the page on non-IE browsers.
258 * @type {HTMLInputElement}
259 * @private
260 */
261 this.hiddenInput_ = /** @type {HTMLInputElement} */ (input);
262
263 /**
264 * The window whose location contains the history token fragment. This is
265 * the window that contains the hidden input. It's typically the top window.
266 * It is not necessarily the same window that the js code is loaded in.
267 * @type {Window}
268 * @private
269 */
270 this.window_ = opt_input ?
271 goog.dom.getWindow(goog.dom.getOwnerDocument(opt_input)) : window;
272
273 var iframeSrc;
274 if (goog.isString(opt_blankPageUrl)) {
275 iframeSrc = goog.html.legacyconversions.trustedResourceUrlFromString(
276 opt_blankPageUrl);
277 } else {
278 iframeSrc = opt_blankPageUrl;
279 }
280
281 /**
282 * The base URL for the hidden iframe. Must refer to a document in the
283 * same domain as the main page.
284 * @type {!goog.html.TrustedResourceUrl|undefined}
285 * @private
286 */
287 this.iframeSrc_ = iframeSrc;
288
289 if (goog.userAgent.IE && !opt_blankPageUrl) {
290 if (window.location.protocol == 'https') {
291 this.iframeSrc_ = goog.html.TrustedResourceUrl.fromConstant(
292 goog.string.Const.from('https:///'));
293 } else {
294 this.iframeSrc_ = goog.html.TrustedResourceUrl.fromConstant(
295 goog.string.Const.from('javascript:""'));
296 }
297 }
298
299 /**
300 * A timer for polling the current history state for changes.
301 * @type {goog.Timer}
302 * @private
303 */
304 this.timer_ = new goog.Timer(goog.History.PollingType.NORMAL);
305 this.registerDisposable(this.timer_);
306
307 /**
308 * True if the state tokens are displayed in the address bar, false for hidden
309 * history states.
310 * @type {boolean}
311 * @private
312 */
313 this.userVisible_ = !opt_invisible;
314
315 /**
316 * An object to keep track of the history event listeners.
317 * @type {goog.events.EventHandler<!goog.History>}
318 * @private
319 */
320 this.eventHandler_ = new goog.events.EventHandler(this);
321
322 if (opt_invisible || goog.History.LEGACY_IE) {
323 var iframe;
324 if (opt_iframe) {
325 iframe = opt_iframe;
326 } else {
327 var iframeId = 'history_iframe' + goog.History.historyCount_;
328 // Using a "sandbox" attribute on the iframe might be possible, but
329 // this HTML didn't initially have it and when it was refactored
330 // to SafeHtml it was kept without it.
331 var iframeHtml = goog.html.SafeHtml.createIframe(this.iframeSrc_, null,
332 {id: iframeId, style: goog.string.Const.from('display:none'),
333 sandbox: undefined});
334 goog.dom.safe.documentWrite(document, iframeHtml);
335 iframe = goog.dom.getElement(iframeId);
336 }
337
338 /**
339 * Internet Explorer uses a hidden iframe for all history changes. Other
340 * browsers use the iframe only for pushing invisible states.
341 * @type {HTMLIFrameElement}
342 * @private
343 */
344 this.iframe_ = /** @type {HTMLIFrameElement} */ (iframe);
345
346 /**
347 * Whether the hidden iframe has had a document written to it yet in this
348 * session.
349 * @type {boolean}
350 * @private
351 */
352 this.unsetIframe_ = true;
353 }
354
355 if (goog.History.LEGACY_IE) {
356 // IE relies on the hidden input to restore the history state from previous
357 // sessions, but input values are only restored after window.onload. Set up
358 // a callback to poll the value after the onload event.
359 this.eventHandler_.listen(this.window_,
360 goog.events.EventType.LOAD,
361 this.onDocumentLoaded);
362
363 /**
364 * IE-only variable for determining if the document has loaded.
365 * @type {boolean}
366 * @protected
367 */
368 this.documentLoaded = false;
369
370 /**
371 * IE-only variable for storing whether the history object should be enabled
372 * once the document finishes loading.
373 * @type {boolean}
374 * @private
375 */
376 this.shouldEnable_ = false;
377 }
378
379 // Set the initial history state.
380 if (this.userVisible_) {
381 this.setHash_(this.getToken(), true);
382 } else {
383 this.setIframeToken_(this.hiddenInput_.value);
384 }
385
386 goog.History.historyCount_++;
387};
388goog.inherits(goog.History, goog.events.EventTarget);

472/**
473 * Starts or stops the History polling loop. When enabled, the History object
474 * will immediately fire an event for the current location. The caller can set
475 * up event listeners between the call to the constructor and the call to
476 * setEnabled.
477 *
478 * On IE, actual startup may be delayed until the iframe and hidden input
479 * element have been loaded and can be polled. This behavior is transparent to
480 * the caller.
481 *
482 * @param {boolean} enable Whether to enable the history polling loop.
483 */
484goog.History.prototype.setEnabled = function(enable) {
485
486 if (enable == this.enabled_) {
487 return;
488 }
489
490 if (goog.History.LEGACY_IE && !this.documentLoaded) {
491 // Wait until the document has actually loaded before enabling the
492 // object or any saved state from a previous session will be lost.
493 this.shouldEnable_ = enable;
494 return;
495 }
496
497 if (enable) {
498 if (goog.userAgent.OPERA) {
499 // Capture events for common user input so we can restart the timer in
500 // Opera if it fails. Yes, this is distasteful. See operaDefibrillator_.
501 this.eventHandler_.listen(this.window_.document,
502 goog.History.INPUT_EVENTS_,
503 this.operaDefibrillator_);
504 } else if (goog.userAgent.GECKO) {
505 // Firefox will not restore the correct state after navigating away from
506 // and then back to the page with the history object. This can be fixed
507 // by restarting the history object on the pageshow event.
508 this.eventHandler_.listen(this.window_, 'pageshow', this.onShow_);
509 }
510
511 // TODO(user): make HTML5 and invisible history work by listening to the
512 // iframe # changes instead of the window.
513 if (goog.History.isOnHashChangeSupported() &&
514 this.userVisible_) {
515 this.eventHandler_.listen(
516 this.window_, goog.events.EventType.HASHCHANGE, this.onHashChange_);
517 this.enabled_ = true;
518 this.dispatchEvent(new goog.history.Event(this.getToken(), false));
519 } else if (!(goog.userAgent.IE && !goog.labs.userAgent.device.isMobile()) ||
520 this.documentLoaded) {
521 // Start dispatching history events if all necessary loading has
522 // completed (always true for browsers other than IE.)
523 this.eventHandler_.listen(this.timer_, goog.Timer.TICK,
524 goog.bind(this.check_, this, true));
525
526 this.enabled_ = true;
527
528 // Initialize last token at startup except on IE < 8, where the last token
529 // must only be set in conjunction with IFRAME updates, or the IFRAME will
530 // start out of sync and remove any pre-existing URI fragment.
531 if (!goog.History.LEGACY_IE) {
532 this.lastToken_ = this.getToken();
533 this.dispatchEvent(new goog.history.Event(this.getToken(), false));
534 }
535
536 this.timer_.start();
537 }
538
539 } else {
540 this.enabled_ = false;
541 this.eventHandler_.removeAll();
542 this.timer_.stop();
543 }
544};

547/**
548 * Callback for the window onload event in IE. This is necessary to read the
549 * value of the hidden input after restoring a history session. The value of
550 * input elements is not viewable until after window onload for some reason (the
551 * iframe state is similarly unavailable during the loading phase.) If
552 * setEnabled is called before the iframe has completed loading, the history
553 * object will actually be enabled at this point.
554 * @protected
555 */
556goog.History.prototype.onDocumentLoaded = function() {
557 this.documentLoaded = true;
558
559 if (this.hiddenInput_.value) {
560 // Any saved value in the hidden input can only be read after the document
561 // has been loaded due to an IE limitation. Restore the previous state if
562 // it has been set.
563 this.setIframeToken_(this.hiddenInput_.value, true);
564 }
565
566 this.setEnabled(this.shouldEnable_);
567};

570/**
571 * Handler for the Gecko pageshow event. Restarts the history object so that the
572 * correct state can be restored in the hash or iframe.
573 * @param {goog.events.BrowserEvent} e The browser event.
574 * @private
575 */
576goog.History.prototype.onShow_ = function(e) {
577 // NOTE(user): persisted is a property passed in the pageshow event that
578 // indicates whether the page is being persisted from the cache or is being
579 // loaded for the first time.
580 if (e.getBrowserEvent()['persisted']) {
581 this.setEnabled(false);
582 this.setEnabled(true);
583 }

587/**
588 * Handles HTML5 onhashchange events on browsers where it is supported.
589 * This is very similar to {@link #check_}, except that it is not executed
590 * continuously. It is only used when
591 * {@code goog.History.isOnHashChangeSupported()} is true.
592 * @param {goog.events.BrowserEvent} e The browser event.
593 * @private

595goog.History.prototype.onHashChange_ = function(e) {
596 var hash = this.getLocationFragment_(this.window_);
597 if (hash != this.lastToken_) {
598 this.update_(hash, true);
599 }

644/**
645 * Gets the location fragment for the current URL. We don't use location.hash
646 * directly as the browser helpfully urlDecodes the string for us which can
647 * corrupt the tokens. For example, if we want to store: label/%2Froot it would
648 * be returned as label//root.
649 * @param {Window} win The window object to use.
650 * @return {string} The fragment.
651 * @private
652 */
653goog.History.prototype.getLocationFragment_ = function(win) {
654 var href = win.location.href;
655 var index = href.indexOf('#');
656 return index < 0 ? '' : href.substring(index + 1);
657};

660/**
661 * Sets the history state. When user visible states are used, the URL fragment
662 * will be set to the provided token. Setting opt_replace to true will cause the
663 * navigation to occur, but will replace the current history entry without
664 * affecting the length of the stack.
665 *
666 * @param {string} token The history state identifier.
667 * @param {boolean} replace Set to replace the current history entry instead of
668 * appending a new history state.
669 * @param {string=} opt_title Optional title used when setting the hidden iframe
670 * title in IE.
671 * @private
672 */
673goog.History.prototype.setHistoryState_ = function(token, replace, opt_title) {
674 if (this.getToken() != token) {
675 if (this.userVisible_) {
676 this.setHash_(token, replace);
677
678 if (!goog.History.isOnHashChangeSupported()) {
679 if (goog.userAgent.IE && !goog.labs.userAgent.device.isMobile()) {
680 // IE must save state using the iframe.
681 this.setIframeToken_(token, replace, opt_title);
682 }
683 }
684
685 // This condition needs to be called even if
686 // goog.History.isOnHashChangeSupported() is true so the NAVIGATE event
687 // fires sychronously.
688 if (this.enabled_) {
689 this.check_(false);
690 }
691 } else {
692 // Fire the event immediately so that setting history is synchronous, but
693 // set a suspendToken so that polling doesn't trigger a 'back'.
694 this.setIframeToken_(token, replace);
695 this.lockedToken_ = this.lastToken_ = this.hiddenInput_.value = token;
696 this.dispatchEvent(new goog.history.Event(token, false));
697 }
698 }

702/**
703 * Sets or replaces the URL fragment. The token does not need to be URL encoded
704 * according to the URL specification, though certain characters (like newline)
705 * are automatically stripped.
706 *
707 * If opt_replace is not set, non-IE browsers will append a new entry to the
708 * history list. Setting the hash does not affect the history stack in IE
709 * (unless there is a pre-existing named anchor for that hash.)
710 *
711 * Older versions of Webkit cannot query the location hash, but it still can be
712 * set. If we detect one of these versions, always replace instead of creating
713 * new history entries.
714 *
715 * window.location.replace replaces the current state from the history stack.
716 * http://www.whatwg.org/specs/web-apps/current-work/#dom-location-replace
717 * http://www.whatwg.org/specs/web-apps/current-work/#replacement-enabled
718 *
719 * @param {string} token The new string to set.
720 * @param {boolean=} opt_replace Set to true to replace the current token
721 * without appending a history entry.
722 * @private
723 */
724goog.History.prototype.setHash_ = function(token, opt_replace) {
725 // If the page uses a BASE element, setting location.hash directly will
726 // navigate away from the current document. Also, the original URL path may
727 // possibly change from HTML5 history pushState. To account for these, the
728 // full path is always specified.
729 var loc = this.window_.location;
730 var url = loc.href.split('#')[0];
731
732 // If a hash has already been set, then removing it programmatically will
733 // reload the page. Once there is a hash, we won't remove it.
734 var hasHash = goog.string.contains(loc.href, '#');
735
736 if (goog.History.HASH_ALWAYS_REQUIRED || hasHash || token) {
737 url += '#' + token;
738 }
739
740 if (url != loc.href) {
741 if (opt_replace) {
742 loc.replace(url);
743 } else {
744 loc.href = url;
745 }
746 }
747};

750/**
751 * Sets the hidden iframe state. On IE, this is accomplished by writing a new
752 * document into the iframe. In Firefox, the iframe's URL fragment stores the
753 * state instead.
754 *
755 * Older versions of webkit cannot set the iframe, so ignore those browsers.
756 *
757 * @param {string} token The new string to set.
758 * @param {boolean=} opt_replace Set to true to replace the current iframe state
759 * without appending a new history entry.
760 * @param {string=} opt_title Optional title used when setting the hidden iframe
761 * title in IE.
762 * @private
763 */
764goog.History.prototype.setIframeToken_ = function(token,
765 opt_replace,
766 opt_title) {
767 if (this.unsetIframe_ || token != this.getIframeToken_()) {
768
769 this.unsetIframe_ = false;
770 token = goog.string.urlEncode(token);
771
772 if (goog.userAgent.IE) {
773 // Caching the iframe document results in document permission errors after
774 // leaving the page and returning. Access it anew each time instead.
775 var doc = goog.dom.getFrameContentDocument(this.iframe_);
776
777 doc.open('text/html', opt_replace ? 'replace' : undefined);
778 var iframeSourceHtml = goog.html.SafeHtml.concat(
779 goog.html.SafeHtml.create('title', {},
780 (opt_title || this.window_.document.title)),
781 goog.html.SafeHtml.create('body', {}, token));
782 goog.dom.safe.documentWrite(doc, iframeSourceHtml);
783 doc.close();
784 } else {
785 goog.asserts.assertInstanceof(
786 this.iframeSrc_, goog.html.TrustedResourceUrl,
787 'this.iframeSrc_ must be set on calls to setIframeToken_');
788 var url = goog.html.TrustedResourceUrl.unwrap(
789 /** @type {!goog.html.TrustedResourceUrl} */ (this.iframeSrc_)) +
790 '#' + token;
791
792 // In Safari, it is possible for the contentWindow of the iframe to not
793 // be present when the page is loading after a reload.
794 var contentWindow = this.iframe_.contentWindow;
795 if (contentWindow) {
796 if (opt_replace) {
797 contentWindow.location.replace(url);
798 } else {
799 contentWindow.location.href = url;
800 }
801 }
802 }
803 }

807/**
808 * Return the current state string from the hidden iframe. On internet explorer,
809 * this is stored as a string in the document body. Other browsers use the
810 * location hash of the hidden iframe.
811 *
812 * Older versions of webkit cannot access the iframe location, so always return
813 * null in that case.
814 *
815 * @return {?string} The state token saved in the iframe (possibly null if the
816 * iframe has never loaded.).
817 * @private
818 */
819goog.History.prototype.getIframeToken_ = function() {
820 if (goog.userAgent.IE) {
821 var doc = goog.dom.getFrameContentDocument(this.iframe_);
822 return doc.body ? goog.string.urlDecode(doc.body.innerHTML) : null;
823 } else {
824 // In Safari, it is possible for the contentWindow of the iframe to not
825 // be present when the page is loading after a reload.
826 var contentWindow = this.iframe_.contentWindow;
827 if (contentWindow) {
828 var hash;
829 /** @preserveTry */
830 try {
831 // Iframe tokens are urlEncoded
832 hash = goog.string.urlDecode(this.getLocationFragment_(contentWindow));
833 } catch (e) {
834 // An exception will be thrown if the location of the iframe can not be
835 // accessed (permission denied). This can occur in FF if the the server
836 // that is hosting the blank html page goes down and then a new history
837 // token is set. The iframe will navigate to an error page, and the
838 // location of the iframe can no longer be accessed. Due to the polling,
839 // this will cause constant exceptions to be thrown. In this case,
840 // we enable longer polling. We do not have to attempt to reset the
841 // iframe token because (a) we already fired the NAVIGATE event when
842 // setting the token, (b) we can rely on the locked token for current
843 // state, and (c) the token is still in the history and
844 // accesible on forward/back.
845 if (!this.longerPolling_) {
846 this.setLongerPolling_(true);
847 }
848
849 return null;
850 }
851
852 // There was no exception when getting the hash so turn off longer polling
853 // if it is on.
854 if (this.longerPolling_) {
855 this.setLongerPolling_(false);
856 }
857
858 return hash || null;
859 } else {
860 return null;
861 }
862 }

866/**
867 * Checks the state of the document fragment and the iframe title to detect
868 * navigation changes. If {@code goog.HistoryisOnHashChangeSupported()} is
869 * {@code false}, then this runs approximately twenty times per second.
870 * @param {boolean} isNavigation True if the event was initiated by a browser
871 * action, false if it was caused by a setToken call. See
872 * {@link goog.history.Event}.
873 * @private
874 */
875goog.History.prototype.check_ = function(isNavigation) {
876 if (this.userVisible_) {
877 var hash = this.getLocationFragment_(this.window_);
878 if (hash != this.lastToken_) {
879 this.update_(hash, isNavigation);
880 }
881 }
882
883 // Old IE uses the iframe for both visible and non-visible versions.
884 if (!this.userVisible_ || goog.History.LEGACY_IE) {
885 var token = this.getIframeToken_() || '';
886 if (this.lockedToken_ == null || token == this.lockedToken_) {
887 this.lockedToken_ = null;
888 if (token != this.lastToken_) {
889 this.update_(token, isNavigation);
890 }
891 }
892 }

896/**
897 * Updates the current history state with a given token. Called after a change
898 * to the location or the iframe state is detected by poll_.
899 *
900 * @param {string} token The new history state.
901 * @param {boolean} isNavigation True if the event was initiated by a browser
902 * action, false if it was caused by a setToken call. See
903 * {@link goog.history.Event}.
904 * @private
905 */
906goog.History.prototype.update_ = function(token, isNavigation) {
907 this.lastToken_ = this.hiddenInput_.value = token;
908
909 if (this.userVisible_) {
910 if (goog.History.LEGACY_IE) {
911 this.setIframeToken_(token);
912 }
913
914 this.setHash_(token);
915 } else {
916 this.setIframeToken_(token);
917 }
918
919 this.dispatchEvent(new goog.history.Event(this.getToken(), isNavigation));

938/**
939 * Opera cancels all outstanding timeouts and intervals after any rapid
940 * succession of navigation events, including the interval used to detect
941 * navigation events. This function restarts the interval so that navigation can
942 * continue. Ideally, only events which would be likely to cause a navigation
943 * change (mousedown and keydown) would be bound to this function. Since Opera
944 * seems to ignore keydown events while the alt key is pressed (such as
945 * alt-left or right arrow), this function is also bound to the much more

947 * unstick itself as the user wiggles the mouse in frustration.
948 * @private
949 */
950goog.History.prototype.operaDefibrillator_ = function() {
951 this.timer_.stop();
952 this.timer_.start();
